Today's mission— the bottleneck

The hard part is perception, not the 3D.

Reading an arbitrary floorplan — any drawing style, country, language, or scan quality — and reconstructing it faithfully is the one thing standing between a person and a magical product. So it's where our effort goes. But it's a bottleneck to remove, not a benchmark to win: if perception ever hurts the product, the product wins.

A user should upload a plan and trust the 3D home is faithful — without verifying every wall, door, or window.

Where we are— what's built

The scaffolding is real.

Experience

Understanding

Building Knowledge Layer

Rooms stop being polygons and start carrying meaning — the semantic groundwork perception builds on.

Perception engine

Style router + gray-poché + gap-bridging

Routes each plan to the right extractor, reads gray-fill CAD walls, and bridges openings into closed rooms — automatically.

Discipline

Held-out benchmark, firewall, data-rights

We only score on plans the system has never seen. A firewall stops leakage; provenance is tracked per plan. This is how trust is earned.
